#### picocom**常用选项**

| **选项** | **说明**                                                     |
| -------- | ------------------------------------------------------------ |
| `-b`     | 设置波特率，例如 `-b 115200`。                               |
| `-l`     | 锁定串口设备，防止其他程序使用该设备。                       |
| `-d`     | 禁用调制解调器控制信号（例如 DTR/DSR）。                     |
| `-f`     | 硬件流控，默认关闭。如果需要启用，使用此选项。               |
| `-y`     | 软件流控，默认关闭。如果需要启用，使用此选项。               |
| `--imap` | 设置输入映射规则，例如 `--imap lfcrlf` 将换行符映射为回车+换行。 |
| `--omap` | 设置输出映射规则，例如 `--omap crlf` 将回车映射为回车+换行。 |
| `--echo` | 启用本地回显，适合某些不回显输入的串口设备。                 |
| `-q`     | 静默模式，隐藏一些启动信息。                                 |
| `-x`     | 退出时立即关闭串口设备（避免重置）。                         |
| `--help` | 显示帮助信息。                                               |

#### **常见场景**

#### 1. **打开串口并通信**

```bash
picocom -b 9600 /dev/ttyUSB0
```

- 打开 `/dev/ttyUSB0` 串口设备，波特率为 9600。

#### 2. **设置波特率、禁用流控**

```bash
picocom -b 115200 -f -y /dev/ttyUSB0
```

- 设置波特率为 115200，并同时启用硬件和软件流控。

#### 3. **添加回车换行**

有些设备需要在发送的每条指令后添加回车或换行符，可以使用 `--omap`：

```bash
picocom -b 115200 --omap crlf /dev/ttyUSB0
```

- 将输出的回车自动映射为回车+换行符。

#### 4. **启用本地回显**

某些串口设备不会回显输入的字符，可以启用本地回显：

```bash
picocom -b 9600 --echo /dev/ttyUSB0
```

#### 5. **退出后不重置串口**

某些设备在退出串口时会因为重置而中断通信，使用 `-x` 禁止重置：

```bash
picocom -b 115200 -x /dev/ttyUSB0
```

------

#### **运行时快捷键**

运行 `picocom` 后，可以使用以下快捷键操作：

| **快捷键**      | **功能**                         |
| --------------- | -------------------------------- |
| `Ctrl-a Ctrl-x` | 退出 `picocom`。                 |
| `Ctrl-a Ctrl-q` | 退出静默模式（或进入静默模式）。 |
| `Ctrl-a Ctrl-c` | 切换本地回显功能。               |
| `Ctrl-a Ctrl-f` | 切换硬件流控功能。               |
| `Ctrl-a Ctrl-y` | 切换软件流控功能。               |
| `Ctrl-a Ctrl-b` | 显示当前波特率和串口状态信息。   |
| `Ctrl-a Ctrl-h` | 显示所有快捷键的帮助信息。       |

------

#### **关闭 `picocom`**

在通信结束后，可以按以下快捷键退出：<u></u>

```
Ctrl-a Ctrl-x
```

------

#### **picocom 和 minicom 的对比**

| **特点**   | **picocom**                  | **minicom**                  |
| ---------- | ---------------------------- | ---------------------------- |
| 配置复杂度 | 简单，直接通过命令行参数配置 | 较复杂，需要进入菜单设置     |
| 功能丰富度 | 轻量，提供基本功能           | 功能丰富，适合复杂通信       |
| 使用场景   | 快速调试串口设备             | 长期和多次使用同一设备的场景 |
